# 4. 테스트(Test) 작성 원칙

테스트 코드는 시나리오를 코드로 검증하는 최종 관문입니다.

1. **1:1 주석 맵핑**: 각 테스트 코드 블록은 시나리오의 특정 구문(`Given`, `When`, `Then`)을 검증하는 목적으로 작성되어야 합니다. 가독성과 추적성을 위해, 검증하는 코드 바로 상단에 **시나리오 구문을 토씨 하나 틀리지 않고 그대로 주석으로 포함**해야 합니다.
2. **완전한 검증**: 시나리오에 작성된 모든 구문은 누락 없이 반드시 테스트 코드로 검증해야 합니다.
3. **동시 업데이트**: 시나리오 파일(`.feature`)의 구문이 업데이트되면, 테스트 코드와 그 주석도 함께 업데이트해야 합니다.
4. **구문 병합**: 특정한 검증 로직이나 액션이 동반되지 않는(단독으로 검증할 내용이 없는) 시나리오 구문이 있다면, 전후의 다른 구문과 병합하여 의미 있는 검증 단위를 만듭니다.
