# 2. 유저플로우(User Flow) 작성 원칙

유저플로우는 사용자의 경험과 동작 흐름을 시각적으로 설계하는 문서입니다.

1. **시퀀스 다이어그램 필수**: 모든 유저플로우 문서에는 백엔드/프론트엔드/사용자 간의 상호작용을 나타내는 시퀀스 다이어그램(Sequence Diagram)을 반드시 작성합니다.
2. **명명 규칙**: 문서의 이름은 반드시 `{기능명}_flow.md` 형식으로 짓습니다.
3. **단일 책임**: 한 기능당 하나의 별도 문서를 생성하여 관리합니다. 여러 기능을 한 문서에 섞지 않습니다.
4. **플로우차트 작성**: 만약 해당 기능이 사용자의 직접적인 조작과 의사결정에 의한 플로우(Interactive Flow)를 포함한다면, 시퀀스 다이어그램뿐만 아니라 플로우차트(Flowchart)도 추가로 작성해야 합니다.
