# 🤖 Antigravity AI-BDD-TDD Environment

<CRITICAL_RULE>
당신은 이 프로젝트에서 코드 작성 전 반드시 BDD-TDD 워크플로우를 강제로 준수해야 하는 시스템입니다.
어떤 상황에서도 사용자의 "명시적인 승인" 없이 다음 단계로 임의로 넘어가는 것은 허용되지 않습니다.
특히 [설계/테스트 작성] 후 승인 대기 없이 [프로덕션 코드 작성]을 진행하면 AI로서의 규칙 위반 및 "시스템 실패(Critical Failure)"로 간주됩니다.
</CRITICAL_RULE>

## 1. 프로젝트 설정
- 본 프로젝트는 `.agents/rules/` 에 명시된 BDD-TDD 절차를 100% 준수합니다.
- 코드를 변경하기 전에 항상 기획 문서(예: `docs/requirements/rtm.md` 또는 `.agents/docs/requirements/rtm.md`)를 먼저 확인하고 상태를 갱신하세요.

## 2. 업무 진행 5단계 (요약)
1. **설계**: 요구사항 확인 후 `.feature` 파일 작성 (User Flow / Scenario)
2. **테스트**: 실패하는 테스트 코드 작성 (TDD 시작)
3. **🛑 승인 게이트 (대기)**: 사용자(User)의 명시적 승인 없이 **절대 다음 단계를 진행하지 말고 턴(Turn)을 종료**하세요.
4. **구현**: 테스트를 통과할 최소한의 프로덕션 코드 구현 및 리팩터링
5. **문서 갱신**: 완료 후 기획 문서의 Status 업데이트

상세한 개발 지침과 코딩 컨벤션은 `.agents/rules/` 폴더 내의 각 마크다운 파일에 정의되어 있으니 자동으로 로드하여 숙지하세요.
