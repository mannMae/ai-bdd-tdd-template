# 0. 워크플로우(Workflow) 및 AI 행동 규칙

모든 작업은 아래의 5단계 순서를 엄격히 따르며, 단계의 누락이나 역행은 허용되지 않습니다. 특히 AI Agent는 지정된 구간에서 반드시 멈춰야 합니다.

## 1. 단계별 작업 순서
1. **설계 (Design)**: `rtm.md` 갱신 ➔ 유저플로우(`*_flow.md`) 작성 ➔ 시나리오(`.feature`) 작성
2. **테스트 (Red)**: 시나리오에 기반한 실패하는 테스트 코드 작성
3. **🛑 승인 대기 (Stop & Review)**: (아래 상세 규칙 참고)
4. **구현 (Green)**: 테스트를 통과시킬 최소한의 기능 코드(프로덕션) 작성
5. **완료 및 커밋 (Refactor & Done)**: 전체 테스트 통과 확인 ➔ `rtm.md` Status 'Done'으로 변경 ➔ 관련된 문서와 코드를 묶어서 커밋(Commit)

## 2. AI 강제 정지(Stop) 규칙 - 매우 중요!
*   **테스트 작성 후 대기:** AI는 테스트 코드 작성을 마친 직후, 반드시 작업을 멈추고 사용자에게 "승인을 대기합니다"라고 보고해야 합니다.
*   **구현 금지:** 사용자가 명시적으로 "승인", "시작", "구현해" 라고 허가하기 전까지는 **절대** 다음 단계(기능 코드 구현)로 넘어가거나 코드를 추측하여 짜서는 안 됩니다.

## 3. 커밋(Commit) 규칙
*   기능(Feature)의 구현이 완전히 끝나서 테스트가 통과하면, 작업의 흔적을 남기기 위해 즉시 커밋을 진행해야 합니다. 
*   이때 변경된 문서(`rtm.md`, 시나리오 등)와 코드(테스트, 기능코드)는 하나의 커밋으로 논리적으로 묶여야 합니다.
