# 1. RTM (요구사항 추적 매트릭스) 원칙

모든 프로젝트의 기능 요구사항은 RTM에서 관리하며, 다음의 원칙을 따릅니다.

1. **완벽한 맵핑**: 각 기능별로 연관된 유저플로우 문서, BDD 시나리오 파일, 그리고 실제 테스트 함수(또는 파일)를 서로 맵핑해야 합니다.
2. **현황 표현**: 각 기능의 작업 진행 현황(예: Pending, WIP, Done 등)을 명확하게 표현합니다.
3. **백로그 역할**: 아직 작업하지 않은 예정된 기능들 역시 모두 RTM에 기록하여 백로그 역할을 겸하도록 합니다.
